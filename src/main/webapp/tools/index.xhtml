OCL2PSQL
The Object Constraint Language (OCL) is a textual, declarative language typically used as part of the UML standard for specifying constraints and queries on models. OCL2PSQL implements a novel mapping from OCL to pure SQL that deals with (possibly nested) iterator expressions, without resorting to imperative features of SQL. OCL2PSQL generates SQL queries that can be efficiently executed on mid- and large-size SQL databases.
SSL2PLSQL
Structured Scenario Language (SSL) is a well-defined language for describing specific scenarios for testing data constraints in a precise way. It is a scripting language inspired by the Object Constraint Language (OCL). The SSL2PLSQL implements a mapping from SSL to SQL in Procedural Language (contains loops, functions, etc.). The SSL2PLSQL produces data scenario script that can be executed on MySQL RDBMS.
